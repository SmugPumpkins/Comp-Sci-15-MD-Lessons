# CSE 1010 - Computer Science 1
1. Demonstrate an understanding of what computer science is and what computer scientists study.
    1. Describe computer science as a discipline and explain its main goal.
    2. Explain how algorithms are used to solve problems.
    3. Describe how computer systems are used to test and run algorithms.
    4. Explain how algorithms are translated into programs.
    5. Identify and describe major areas of computer science, including:
        1. theory of computation
        2. algorithms and data structures
        3. programming languages and programming methods
        4. computer hardware and system architecture
        5. human–computer and machine–machine interaction
        6. automata
        7. artificial intelligence
        8. visual and audio rendering
        9. development of information technology applications
    6. Compare computer science, computer engineering, and information technology by looking at differences such as:
        1. theoretical vs. applied work
        2. general vs. specialized focus
        3. exploratory vs. practical goals
    7. Identify and explain common misconceptions about computer science (for example, that it is only programming, done alone, or disconnected from real-world problems).
    8. Explain the role of computer science in an information-based society.
2. Demonstrate an understanding of basic algorithms used to process data in sequence.
    1. Define what an algorithm is and explain its purpose.
    2. Explain how algorithms are used to solve problems involving input, processing, and output.
    3. Compare the iterative and incremental model of software development with the waterfall model.
    4. Demonstrate the analysis and design stages of the Systems Development Life Cycle (SDLC) using tools such as:
        1. flowcharts
        2. pseudocode
        3. input–process–output (IPO) charts
    5. Design and explain common core algorithms, including:
        1. accumulation (keeping a running total)
        2. calculating an average (mean)
        3. finding minimum and maximum values
3. Explain and demonstrate the principles of structured programming.
    1. Explain why structured programming is used.
    2. Describe why modern programming avoids the use of GOTO statements.
    3. Identify and use the three fundamental control structures:
        1. sequence
        2. selection (decision making)
        3. iteration (looping)
4. Demonstrate an understanding of programming languages and how they are used.
    1. Describe how programming languages represent data, including:
        1. binary and hexadecimal systems
        2. common data types
        3. how data is stored in memory
    2. Explain key characteristics of programming languages, including that they:
        1. resemble simplified natural language
        2. have evolved alongside algorithms and hardware
        3. follow the input–process–output (IPO) model
    3. Demonstrate how programming languages are used during the coding stage of the SDLC by converting algorithms into working programs.
5. Explain the purpose, evolution, and structure of a von Neumann computer system.
    1. Create and label a block diagram of a typical von Neumann machine.
    2. Identify common devices associated with each part of the system.
    3. Explain how data moves through the computer system under the control of a program.
# CSE 1110 - Structured Programming 1
1. Demonstrate introductory structured programming skills by planning and writing sequential algorithms.
    1. Explain what an algorithm is and why it is used.
    2. Analyze simple algorithms and describe the task or tasks they perform.
    3. Determine whether a problem can be solved using an input–process–output (IPO) approach.
    4. Break a problem into input, processing, and output components.
    5. Identify which data is already available to the program and which data must be provided as input.
    6. Arrange inputs, processing steps, and outputs in the correct order so that:
        1. processing occurs only after all required input is available
        2. output occurs only after processing is complete
    7. Write algorithms in a clear, standard format (for example, pseudocode or structured charts).
    8. Test algorithms using appropriate data to check for both correct results and possible failures.
    9. Revise and improve algorithms based on testing results.
2. Demonstrate the ability to translate algorithms into working programs.
    1. Describe the purpose of key parts of a programming development environment, including:
        1. source code editor
        2. compiler or interpreter
        3. program executor
        4. debugger
    2. Compare an integrated development environment (IDE) with a setup that uses separate tools (such as a text editor and command-line compiler).
    3. Use the key components of a programming development environment to write and run programs.
    4. Convert algorithms into code using an appropriate programming language while:
        1. maintaining a clear input–process–output (IPO) structure
        2. including internal and external documentation (such as comments and planning notes)
        3. using appropriate data types (integers, real numbers, characters, strings)
        4. using variables and constants correctly
        5. supplying data using literals and input commands
        6. using assignment, arithmetic, string concatenation, and interpolation operators as needed
        7. displaying results using output commands in a clear way
    5. Test programs using appropriate data to identify both correct behavior and errors.
    6. Revise the program as needed based on testing results.
3. Analyze program results and debug errors.
    1. Compare the program’s output with the intended results of the original algorithm.
    2. Use test data and debugging techniques to find and fix errors, including:
        1. run-time and syntax errors that prevent the program from running
        2. logic errors where the program runs but produces incorrect results
    3. Modify the algorithm or code as needed to correct errors and improve accuracy.
# CSE 1120 - Structured Programming 2
1. Demonstrate the ability to plan for structured programming challenges that incorporate selection and iteration structures
    1. Determine whether a problem can be solved with an algorithm using the input–process–output (IPO) model.
    2. Decide if the problem contains more than one IPO module.
    3. Break the problem into smaller modules and identify the input, processing, and output for each.
    4. Identify what data the program already has, what data the user must provide, and group those inputs logically using suitable control structures.
    5. Identify the processing steps needed for the solution and group them logically using suitable control structures.
    6. Use common algorithm patterns when needed, such as accumulation or finding maximum/minimum values.
    7. Identify the required outputs and group them logically using suitable control structures.
    8. Arrange all inputs, processes, and outputs in a logical order so that processing only happens when all needed data is available, and output only happens after processing is complete.
    9. Write the final algorithm in a clear, standard format (e.g., pseudocode or a structured chart).
2. Demonstrate the ability to take an algorithm and convert it to a programming language. Run the program and see the results.
    1. Maintain a clear input–process–output (IPO) structure in the algorithm.
    2. Include appropriate internal and external documentation. This can include comments in your code that explain how it works and planning you’ve done previously.
    3. Use basic data types correctly (integer, real/float, character, string, Boolean).
    4. Use variables and constants appropriately to store data.
    5. Use literals (setting values manually or assigning them to variables in the code) and input commands (allowing the user to set values through input or interaction) to supply data for processing.
    6. Use the appropriate operators to work with data (for example, assigning values, doing math, comparing values, combining text, or inserting values into strings, etc.) whenever the task requires it.
    7. Use common algorithm patterns when needed, such as accumulation or finding maximum/minimum values.
    8. Use suitable selection and looping structures to avoid unconditional jumps, including:
        1. nested conditional blocks
        2. nested loops
    9. Use output commands (methods or operators) to display results in a clear, formatted way.
3. Run the program. Determine if there are any errors and fix them so that the program runs as expected.
    1. Use suitable test data to check the program. This can include user inputs or literals that you manually program to check if things work as expected.
        1. Use debugging techniques to track and correct run-time or syntax errors. These are errors that occur when a program will not run at all until the code is correct.
        2. Use debugging techniques to find and fix logic errors. These are errors where the code does run, but produces an unexpected result due to the logic used in the program.
# CSE 1210 - Client Side Scripting 1
1. Demonstrate an understanding of the general architecture of the Internet as it relates to client-side scripting.
    1. Explain and demonstrate the client/server nature of the Internet, including:
        1. Describe the Internet as a collection of interconnected networks.
        2. Illustrate the client/server relationships among workstations, local networks, regional networks, and the Internet backbone.
        3. Describe how servers, routers, switches, workstations, and other hardware support client/server communication.
        4. Describe the Internet Protocol Suite (TCP/IP) in general terms and explain how it enables data transfer between clients and servers.
        5. Illustrate the client/server interaction that occurs when a user requests an Internet service.
        6. Identify and describe at least three Internet services that rely on client/server relationships.
    2. Explain and demonstrate the client/server nature of the Web, including:
        1. Describe the hypertext-based nature of the Web.
        2. Describe the relationship between the Web and the Internet.
        3. Explain why the Web can be viewed as a network of linked documents.
        4. Describe the client/server relationship between user agents (browsers) and origin servers.
        5. Describe how Hypertext Transfer Protocol (HTTP) supports client/server communication.
        6. Illustrate the general flow of information when a browser communicates with a web server.
        7. Explain how HTTP is used to help protect data during transmission.
        8. Describe the development of the Web using the Web 1.0, Web 2.0, and Web 3.0 models.
        9. Compare the characteristics of Web 1.0, Web 2.0, and Web 3.0.
2. Demonstrate an understanding of the general nature and purpose of Internet-oriented markup languages.
    1. Describe the role markup languages play on the Web.
    2. Compare markup languages and scripting languages.
    3. Describe the development and relationships among Internet-oriented markup languages, including:
        1. the relationship between SGML, XML, HTML, XHTML, and DHTML
        2. at least two specialized Internet markup languages
3. Design, write, and debug documents using an appropriate Internet markup language.
    1. Use an appropriate coding environment to create and test markup documents.
    2. Design a markup document by:
        1. determining the purpose of the document
        2. organizing content into sections or pages
        3. identifying the content for each section
        4. planning how the content will be displayed
        5. identifying required anchors and hyperlinks
    3. Translate design plans into working hypertext documents by using tags, attributes, and hyperlinks to:
        1. structure the parts of the document
        2. display text and visual content in multiple formats
        3. create structured elements such as lists, tables, and frames
        4. create both text-based and image-based hyperlinks
    4. Compare the completed document with the original design and revise as needed by:
        1. using debugging techniques to identify differences between the design and implementation
        2. modifying the design or the document so both match the intended outcome
